BONUS[ tips ]:

Wrong Branch Uncommited Work Fix using Git Stash:
additional resoursce: Git Stash - Repo not in correct branch fix.
>
wrong branch, uncommited changes - Typing git status shows uncommited work, trying to switch branch with git checkout feature fails.
stash changes off to the side - Type git stash in terminal to place the uncommited work off to the side and out of the wrong branch.
switch to correct branch - Check with git status to make sure the uncommited work has been stashed, if so use git checkout BRANCH-NAME.
retreive the stash and commit - Type git stash pop in terminal to pop the stashed work into the correct branch. Finish the commit, voila!

merge rejected in terminal - Type git pull in terminal to pull down changes that's currently not on your local machine.
local code indication - THe code highlighted in 'HEAD (Current Change)' is code available on your local machine, your changes.
pulled code indication - the code witht he hash#########... (Incoming Change) is the code that's conflicting with yours.
fix merge conflicts - You can Accept current change(your code), Accept incoming change(Conflicting Code), or fix the hightlighted code manually.

BONUS[ tips ]:

Avoiding Merge Conflicts:
--- Deal with any uncommitted work before trying to merge.
--- Collaborators should avoid working on the same file at the same time.
--- Try to avoid adding code directly into main. New ideas should be explored in a different branch first and then merged.
